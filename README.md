# Corne 高速输入布局

[English](README.en.md)

![Default Layer](default_layer.png)

这是我自用并分享的 Corne 三层输入布局，目标是让 42 键分体键盘在日常输入、符号输入和导航操作之间切换得更快、更顺手。

这套布局的核心思路是：默认层负责高频字母输入，符号层集中处理标点和编程符号，功能层放置数字、方向键和常用系统控制。大多数操作都可以通过拇指按住层键完成，减少手指离开主键区的次数。

层切换也尽量保留“哪只手按层键，就优先触发哪一侧键盘扩展能力”的感觉。左侧层键更偏向左手区域的符号输入，右侧层键更偏向右手区域的数字、导航和功能操作，让临时单手操作更直觉。

为了保持快速输入时的确定性，这套布局只使用 3 个实际输入层，不依赖 Tap Dance 和 Combo。Tap Dance 在高速打字时需要等待固件判断单击、双击或长按，会打断输入节奏；Combo 在输入速度较快时更容易误触发。因此，这个布局优先选择直接、稳定、低延迟的层切换。

## 布局目标

- **快速输入**：常用字母、空格、退格和修饰键都放在容易触达的位置。
- **三层覆盖日常操作**：默认层、符号层、功能层分别处理不同输入场景。
- **拇指驱动层切换**：通过 `MO(1)` 和 `MO(2)` 进入符号层和功能层。
- **分侧层切换**：左手触发左侧扩展，右手触发右侧扩展，尽量保留单手操作的自然感。
- **适合编程**：括号、运算符、数字和方向键集中在独立层，减少组合键负担。
- **不使用 Tap Dance / Combo**：避免等待判定和高速输入时的误触发。
- **Vial 友好**：布局文件为 `layout.vil`，可用于 Vial/VIA 相关工作流。

## 三层设计

### 1. 默认层

![Default Layer](default_layer.png)

默认层用于主要文字输入。字母采用左右手分布，拇指区放置 `LAlt`、`LGui`、`MO(1)`、`MO(2)`、`Space`、`RAlt` 等高频按键。

设计重点：

- 左右手主键区负责字母输入。
- `MO(1)` 进入符号层。
- `MO(2)` 进入功能层。
- 层键按左右手分工放置，尽量让触发层的那只手获得对应侧的扩展能力。
- `Backspace`、`Space`、修饰键靠近拇指或边缘，便于快速触发。

### 2. 符号层

![Symbol Layer](symbol_layer.png)

符号层主要用于标点、括号和编程符号输入。按住 `MO(1)` 即可进入此层，松开后返回默认层。

设计重点：

- 数字上排映射为常见符号，例如 `! @ # $ % ^ & * ( )`。
- 括号、反斜杠、加减号、等号集中在左手区域。
- 适合代码输入、Markdown、命令行和日常标点输入。

### 3. 功能层

![Function Layer](functuion_layer.png)

功能层用于数字、方向键、Esc、Enter 和个人高频功能键。按住 `MO(2)` 即可进入此层。

数字区的布局尽量沿用普通键盘的数字排逻辑，放在默认层 `QWER` 对应的那一列附近。这样从标准键盘迁移到 Corne 时，不需要重新建立一套完全陌生的数字位置记忆，输入数字时更容易靠原有肌肉记忆过渡。

设计重点：

- 数字键集中在上排，并贴近普通键盘数字排的相对位置。
- 方向键放在右手主键区，便于编辑文本时快速移动光标。
- `F12` 是我的输入法切换键，`F8` 是我的 tmux prefix，因此放在功能层中快速触发。
- `Esc`、`Enter` 等编辑控制键也放在功能层，减少默认层负担。
- 适合代码编辑、窗口操作和系统快捷键。

## 速度取舍

这套布局的重点不是把所有高级功能都塞进 42 键，而是让高频输入路径尽量直接。

- 不使用 Tap Dance：快速打字时不需要等待 Tap Dance 判定，按键反馈更直接。
- 不使用 Combo：减少高速输入时因为相邻按键同时触发导致的误操作。
- 只使用 3 层：默认层、符号层、功能层覆盖主要输入需求，记忆成本更低。

## 使用方法

1. 连接 Corne 键盘。
2. 打开 [Vial Web](https://vial.rocks/)。
3. 在页面中连接设备。
4. 选择 `File` -> `Load saved layout`。
5. 加载本仓库中的 `layout.vil`。
6. 根据自己的手型、输入习惯和系统快捷键继续微调。
